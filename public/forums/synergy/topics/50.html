  7. Greg Hurrell 2004-11-24T06:28:22Z

    Quote: Excellent! I've been wanting this for a long time. Nice to see Synergy now idling at 0.0%. I'm interested in the internal changes to iTunes that enable this - does iTunes now post notifications to OS X's notification service?     It's called NSDistributedNotificationCenter. If you look at the WinSwitch source code you can see the kind of reverse engineering one needs to do to figure out these things (thankfully, not too much!).

    Quote: I'm guessing Apple finally decided to add this functionality to iTunes because their iTunes widget in Tiger needs it. It would be a bit of an ugly hack if they had to constantly poll iTunes. (Not saying that Synergy has been an ugly hack until now - far from it, it's just that Apple has the power to implement it more elegantly so it's not surprising they chose to do so.)

    I think you're dead right. I also think they're using it for the iChat/iTunes integration in Tiger. Needless to say, I'll be taking full advantage of all this for Synergy Advance, which will include Dashboard support.

    And as for "ugly hacks", well, it wasn't an ugly hack, but it was a kludge. I am very glad to be rid of it. And from the feedback, the users love the increased performance. Next version should have even more performance. In fixing the 10.2.8 bug I've already cut remaining communications with iTunes in half again.
