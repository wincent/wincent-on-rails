  5. Greg Hurrell 2005-04-16T07:08:17Z

    You're right that it doesn't make sense to look for new artwork for every song on an album. This is a problem that Synergy has always had. How does it know "what is an album"? It can only assume that if a track has the same album title and the same artist then it is the same album. This has caused problems for compilation albums because the artists are often different. In this case, Synergy should ignore the artist name and perform the comparison on the basis of album name only.

    But there are other problems with that idea because some people have different art embedded in tracks even though they are in the same album... For this reason I am going to implement some preferences in Synergy Advance so people can choose how their art gets managed (ie. the strategy used for identify what is an album).

    The 1.8 release of Synergy does exacerbate this problem because composer is now part of the cover art identifier. As you've pointed out you'll see this when a band has different composers for tracks on the same album.

    I do appreciate the comments but given the comments should really go in the bug database so that all the information relating to the problem is in one place. I'm going to lock this thread. Please add any additional comments there.
