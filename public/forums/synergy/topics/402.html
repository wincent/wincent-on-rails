  2. ZoomAddict 2009-01-31T13:17:20Z

    Didn't find the cause but poked around the website and found this workaround which works:

    ---------- "Launch Synergy at login" checkbox in the Preference Pane no longer works for Synergy Classic. The workaround is to navigate to ~/Library/PreferencePanes/Synergy.prefPane, ctrl+click, choose Show Package Contents from the contextual menu, navigate to Contents/Resources/Synergy and drag this icon to the Startup Items in the Accounts preference pane. ----------

    This issue (failure of the "Launch at Startup" feature) in Synergy Classic was repeatable (same issue on both the Mac Pros tested here). If you are still doing fixes to the 3.2.x version this bug needs to be quashed. Thanks!

  3. Greg Hurrell 2009-01-31T13:19:17Z

    The unchecking "Launch at login", "Apply", rechecking, "Apply" should have worked.

    If you want to try adding it manually to the startup items the item you have to add is not ~/Library/PreferencePanes/Synergy.prefPane but ~/Library/PreferencePanes/Synergy.prefPane/Contents/Resources/Synergy.app.

    There might be something else going on here. Any error messages printed to the console? Is it possible that it is loading but you just can't see it because your menu bar is too full of other items?

  4. Greg Hurrell 2009-01-31T13:24:29Z

    The "not launching at login" issue has come up a few times over the years (Synergy first came out in 2002, remember). Occasionally in an OS upgrade things would change in some subtle way and it would stop working. I don't think there were any "bugs" as such in the way Synergy added itself to the login items, but over the years I have had to implement a lot of workarounds to make sure that it got added when required.

    It's actually been surprisingly complicated for what should be such a simple feature.

    At the time of writing, I'm not aware of any real problems with 3.2.1 and the latest version of Mac OS X, and your report is an isolated one. If it were a general issue I would expect literally dozens of reports from different people. But still, it's something to keep an eye on to see if there's been yet another change in the way login items works which is causing problems.
