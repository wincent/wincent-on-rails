Bug #322: Playing a track from a playlist doesn't switch to that playlist

Kind bug
Product Synergy Advance
When Created 2005-10-25T22:15:50Z, updated 2006-05-07T07:33:23Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

"[W]hen I use the playlist feature with the menubar, I am successfully taken to the song I want. But if I click the next or previous button, it takes me back to whatever playlist I was on earlier. For instance, let's say I'm currently in a Beatles playlist playing "Let it be." If I decide I'm more in a Pink Floyd mood, I'll go to the playlist from the menubar and choose a Floyd song. But since I'm in that playlist, I might want to jump ahead to a different Floyd song. If I click the forward button, it goes back to the Beatles playlist and plays the next song that was in that list. I'd prefer to just have the buttons work with the current playlist."

Good point. Synergy Advance merely tells iTunes "play song X from playlist Y", but from what you're saying perhaps it should tell iTunes "switch to playlist Y and play song X".

"I can see advantages to the current Synergy approach as well as the one I just described. (What I described can be done. Perhaps Synergy could offer both options and let the user choose?"

Agreed. I'll try to add this as an option in the "Advanced" preferences pane, defaulting to the behaviour that you describe. The other thing I could do is allow the user to press the Option key as a modifier to temporarily toggle the behaviour; in other words, you could have it set up to normally do a proper playlist switch whenever you select a song from a playlist, but you could temporarily override the behaviour by pressing the Option key, thus making it play the song but not actually switch from your old playlist.

Thanks for submitting the report.
