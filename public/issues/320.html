Bug #320: Menu bar buttons can move to the left when changing button sets

Kind bug
Product Synergy
When Created 2005-10-25T22:00:28Z, updated 2006-07-05T02:06:50Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

"With Synergy, when I switch to a different theme for my menubar controllers, the position of Synergy shifts in my menubar. I have Synergy open on startup, and the controllers are on the right side of the menubar. When I change to something else in Synergy, it shifts to the left. I find this annoying. Any way to make it stay put?"

Unfortunately there is no way to make it stay put at this stage. Synergy has no control over where it appears in the menu bar; the system will always put newly added items to the left, so the load order is what determines the order in which things appear. When you change button sets Synergy is actually removing itself from the menu bar and re-adding itself with the new set, which is why it will always appear on the left after a set change.

The only way to change this will be to change the way Synergy handles set changes internally. I did it this way because different button sets can be of different widths, and the simplest way to change the width without getting any unwanted side effects or visual glitches was to just remove Synergy and re-add it. But thanks for submitting the report; if I can work out a way to change things in place without any glitches then I'll definitely do it.
