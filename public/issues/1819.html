Bug #1819: command-t crashes after 50 or so invocations

Kind bug
Product Command-T
When 2011-05-17T15:59:41Z
Status closed
Reporter anonymous
Tags no tags

Description

I'm running stock Ubuntu Natty. I'm sure I'm compiled against the system ruby. Command-T almost always works but, once in a while, it crashes. It's not hard to reproduce:

nmap <silent> <C-Space> :CommandT<CR>

Now hold down control and rapidly type SPACE C SPACE C SPACE C rapidly about 50 times. After a while you'll see:

Vim: Caught deadly signal SEGV
Vim: Finished.
