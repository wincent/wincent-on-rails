Feature request #184: REQUEST: Add more states and animation to buttons, and use Photoshop button files

Kind feature request
Product Synergy Advance
When Created 2005-04-16T05:17:31Z, updated 2006-07-05T02:05:08Z
Status open
Reporter Eerk
Tags no tags

Description

Although this will definitely cause horrible button sets to appear, I think that, when used subtle and with caution, additional states and animation (play!) will enhance the look and feel of button sets. This is an addition to the request in #165. For another application I worked on the interface for, the developers had a interface button element that used layered PhotoShop files for input, including transparency. By naming the layers with specific states, this one file is able to serve all the states needed. A very convenient way to design buttons! This could even include animation by appending "#1", "#2", etc. to one state's name.

  1. Greg Hurrell 2005-04-16T06:47:05Z

    Interesting ideas. I agree the animations could potentially become quite horrible, but there's also potential for good in there.

    As for the Photoshop idea I don't think it would be a very high priority for me to implement that feature because there's no native support for opening PS files within Cocoa (at least, as far as I know). I think one of the real strings of Mac OS X's bundle system is that you can browse a bundle in the file and see all its components without having to rely on other applications (especially very expensive ones like Photoshop which many people can't afford). Of course, I'm always ready for someone to come and change my mind...

    So for now we have the core proposal: an animation-capable button set format.

    And some options for how these animations could be stored:

    1. Photoshop layers 2. Animated GIFs 3. Numbered files 4. QuickTime movies
