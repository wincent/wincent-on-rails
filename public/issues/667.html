Bug #667: Hide "communication rate" slider in preferences UI

Kind bug
Product Synergy
When Created 2008-04-13T07:50:17Z, updated 2008-05-28T12:35:17Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

The "Advanced" tab of the preferences includes a slider for altering the communication rate between Synergy and iTunes, but it is ghosted (disabled) when running versions of iTunes older than 4.7.

I occasionally get bug reports claiming that the slider isn't working (even though there is a tool tip that explains why the slider is ghosted); so although things are working perfectly it is arguably a UI design bug that the interface is too subtle and allows this kind of misunderstanding to arise.

In 3.5 I can afford to hide the slider because 3.5 is Leopard-only and it is not even possible to run a pre-4.7 iTunes on Leopard.

  2. Greg Hurrell 2008-04-15T14:00:32Z

    There are actually a couple issues here:

    • one is that there's a bug in versions of iTunes since 7.6.1 which often prevents the artwork from coming through (see bug #668 for more details); so under these circumstances Synergy will fall back to looking on the Internet and as you've seen it sometimes gets undesirable results.
    • the other issue is that once Synergy has downloaded art for a particular track it prefers the downloaded art over the art in iTunes — this was originally for efficiency reasons because it is much quicker to read a file off the disk than it is to retrieve artwork from iTunes — but there is a ticket open for this (see feature request #54) which proposes changing that prioritization.
  3. Greg Hurrell 2008-04-29T10:24:39Z

    Ok, I've now removed the slider. This change will become available in the next release in the 3.5 series (just released 3.5a7 today, so we're talking 3.5a8 or 3.5b).
