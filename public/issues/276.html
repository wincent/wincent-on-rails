Bug #276: Work around scrolling glitches in BBEdit

Kind bug
Product Synergy Advance
When Created 2005-07-27T00:39:00Z, updated 2006-03-29T07:53:31Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

If Synergy Advance displays a bezel and then fades it out, the NSWindow "close" method is eventually called, but it appears that the window is not being fully ordered out from the display.

This manifests itself if you open a Carbon app like BBEdit (well, BBEdit is the only app in which I've seen it) and try to scroll a document. There are visible artefacts which line up with where the bezel was prior to fading. I'll try to work around this defect for public preview release 0.3.
