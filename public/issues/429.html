Feature request #429: REQUEST: Automatically change to random button set at user-definable interval

Kind feature request
Product Synergy
When Created 2006-06-02T09:30:35Z, updated 2006-09-04T08:43:03Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From a post the forums:

"I would like to have better control over my buttons. I would like them to be able to change at specifiied intervals (such as every hour, every day, ect)."

http://wincent.com/a/support/forums/showflat.php?Number=1004

My reply:

You are probably aware that you can already instruct Synergy to use a different random button set each time it is launched. Look in the advanced tab of the preferences. When this option is in effect the act of hiding and redisplaying the menu bar buttons causes them to jump to a new, randomly-selected set. You can define a hot key for this action (hiding/showing) which can make cycling through button sets relatively rapid.

All the same, I think your feature request is a good idea. I would add this as a pop-up menu in the preferences. I think it makes sense to implement a hot key to cycle to the next button set as well.

In the meantime you could as a workaround take advantage of the existing feature in Synergy and whip up an AppleScript or a cron job that could quit and relaunch Synergy once per hour, once per day (or whatever) so as to cause the button sets to rotate.
