Bug #434: Queue for Album Cover Downloads: Memory -> File

Kind bug
Product Synergy
When Created 2006-06-19T02:59:57Z, updated 2006-07-05T02:08:05Z
Status open
Reporter Jason Harper
Tags no tags

Description

I regularly use synergy on a computer which is not connected to the internet; however I do regularly synchronise this computer with one which is always connected to the internet but on which I rarely run synergy. I'm looking to try and ensure that album covers are downloaded on the one connected to the 'net (even if the song was played on the other computer).

Is it possible to introduce a feature where the 'queue' for album cover requests is stored in a file? As such, in synchronising files between computers, I could synchronise that one and the album covers would be downloaded on those occasions on which I did connect to the internet.

Thanks in advance - Synergy is one shareware programme that I am incredibly satisfied with!

Jason

    Thanks for writing this one up, Jason. Given that the search strings are just that, strings, they should be fairly easy to store on disk.

    I guess we'd really need two queues for this one to work well, though. One, the in-memory queue, would have priority. The on-disk queue would only be used if the in-memory one was empty. On quitting Synergy the in-memory queue would be appended to the on-disk queue.
