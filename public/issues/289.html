Feature request #289: REQUEST: Bring menu and controls to front when obscured by application menus

Kind feature request
Product Synergy Advance
When Created 2005-08-17T18:26:28Z, updated 2006-07-05T02:06:16Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

"Is there any way to make the synergy control not disappear when another program's menu items are too many to fit? The Mozilla Thunderbird takes over my menu bar almost completely, and that's when I need Synergy the most. :/"

My reply:

"Not really. You can try to make Synergy load before other NSStatusItems so that it will appear as far to the right as possible, but that still won't help in all cases. About all I can suggest is trying to get by with Synergy's hot keys in those circumstances."

But there are a few ideas that spring to mind that could be used to address this problem:

(1) A hot key that could be used to bring the global menu and/or menu bar control buttons to the front. Basically, I would have to throw up a window which would float in front of the menu bar (or just below it) whenever the hot key was pressed and/or pressed-and-held. In that window I could put the global menu and/or menu bar control buttons etc. After a user-configurable interval the window would fade out or disappear.

(2) Throwing the mouse into one of the screen corners could be configured to trigger the same thing.

(3) Dumping the NSStatusItem API and moving to NSMenuExtra, which would allow the user to reposition Synergy within the menu bar by holding the Command key and dragging. I definitely won't be going down this path because NSMenuExtra is a undocumented, private API that is not supported by Apple.
