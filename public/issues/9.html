Bug #9: Switching playlists brings iTunes to the front

Kind bug
Product Synergy
When Created 2004-11-07T01:42:36Z, updated 2010-08-12T09:47:20Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

From a user email: "when changing playlists i'd really like itunes to stay closed if it is closed or hidden if it is hidden. the whole point of using a menu item to swap playlists is so that i don't have to have itunes on the screen.."

I'd already discussed this with another user, and this was the dialogue we had:

User: "iTunes brought to front when switching playlists (prefs set to not bring to front)."

Wincent: "I can't reproduce this bug. Is the iTunes main window open when you try this?"

User: "yes, it is, but it's minimized.

of course, itunes must be open for synergy to work. if i start a song from synergy w/ itunes closed, itunes opens and i have to close the active window but not the app. then i can choose songs w/o itunes popping up again.

if itunes is minimized, switching playlists causes itunes to open in the finder. then i have to close it. minimizing it doesn't work."

Wincent: 'Ok, thanks. Thanks to the extra information I can reproduce the bug.

Personally, I think this is a bug in iTunes. Consider this: Synergy is only telling it to switch playlists, it's not making a different type of request depending on whether iTunes is minimized. In fact, it doesn't even know that iTunes is minimized. I view this as inconsistent behaviour on the part of iTunes:

1. When iTunes window is in "normal" state: doesn't come to front 2. When iTunes application is hidden: doesn't come to front 3. When iTunes window is in "mini" mode: doesn't come to front 4. When iTunes is minimized in dock: comes to front

Not sure if there's anything I can do to fix this.'
