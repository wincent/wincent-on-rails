  2. Greg Hurrell 2008-01-08T01:59:07Z

    Ok, found out the problem here and fixed it. Adding the fixed-in-prerelease keyword.

    Looks to me to be a bug in the new Interface Builder that comes with Leopard. New projects always default to English, but it looks like if you create a new nib file then it defaults to language on which you're running. This is a regression, in my opinion, seeing as it's a discrepancy from the behaviour in 10.4, (and 10.3, 10.2).

    Somewhat bizarrely, not all of the menus were in Spanish, only some of them. I don't actually think that's the fault of Interface Builder, however; I think I edited those items and translated them to English without even thinking about it (one of the problems with being bilingual, sometimes you don't even realize what language you're speaking/reading/writing). I just created a new nib and confirmed that everything was in Spanish, so that's definitely what happened.

    Keep your eyes out for a new release which includes the fix, 3.5a4, very soon.
