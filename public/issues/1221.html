Bug #1221: Update Apache redirects to point at wincent.com instead of rails.wincent.com

Kind bug
Product wincent.com
When Created 2009-02-11T12:04:02Z, updated 2009-02-12T05:03:18Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

Now that the app is being served out of wincent.com, we can update some old redirects which Apache is still pointing at rails.wincent.com.

$ grep -B 4 rails wincent.com.conf 
    RewriteRule ^/?(.*) http://wincent.com/a/products/bansshee/$1 [redirect=permanent,last]

    # bugs.wincent.com   
    RewriteCond %{HTTP_HOST} ^bugs.wincent.com$ [nocase]
    RewriteRule .? http://rails.wincent.com/issues/ [redirect=permanent,last]

    # colaiuta.net, www.colaiuta.net
    RewriteCond %{HTTP_HOST} ^colaiuta.net$ [nocase,OR]
    RewriteCond %{HTTP_HOST} ^www.colaiuta.net$ [nocase]
    RewriteRule .? http://rails.wincent.com/blog/ [redirect=permanent,last]

    # forums.wincent.com
    RewriteCond %{HTTP_HOST} ^forums.wincent.com$ [nocase]
    RewriteRule .? http://rails.wincent.com/forums/ [redirect=permanent,last]
--

    # wincent.org, www.wincent.org
    RewriteCond %{HTTP_HOST} ^wincent.org$ [nocase,OR]
    RewriteCond %{HTTP_HOST} ^www.wincent.org$ [nocase]
    RewriteRule .? http://rails.wincent.com/blog/ [redirect=permanent,last]
--

  # catch everything under /a/support/bugs/
  <Directory /home/wincent.com/public_html/a/support/bugs>
    RewriteEngine on
    RewriteRule ^/?(.*) http://rails.wincent.com/issues/? [redirect=permanent,last]
--

  # catch everything under /a/support/forums/
  <Directory /home/wincent.com/public_html/a/support/forums>
    RewriteEngine on
    RewriteRule ^/?(.*) http://rails.wincent.com/forums/? [redirect=permanent,last]
--

  # catch everything under /a/support/kb/
  <Directory /home/wincent.com/public_html/a/support/kb>
    RewriteEngine on
    RewriteRule ^/?(.*) http://rails.wincent.com/wiki/? [redirect=permanent,last]
