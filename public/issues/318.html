  4. Greg Hurrell 2005-11-10T05:18:08Z

    Ok, now that I've seen your video and examined the code I've found the source of the code.

    The problem is that Synergy Advance is updating iChat because you have the checkboxes checked to update "When iTunes track changes" and "When iTunes player state changes", regardless of whether or not you have the "Update iChat status based on current track" checkbox turned on. The correct behaviour would be to ignore the former checkboxes whenever the latter is unchecked. I will go ahead and correct this right now for the next release.

    In the meantime, the workaround is to uncheck the offending checkboxes. Adding the "fixed-in-prerelease" keyword.
