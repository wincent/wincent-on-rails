Bug #120: Floater caches/uses incorrect artwork

Kind bug
Product Synergy
When Created 2005-02-19T08:58:52Z, updated 2005-03-04T06:59:28Z
Status closed
Reporter Dan
Tags no tags

Description

The display floater shows artwork that does not match what I have assigned to the track in iTunes. I suspect either Synergy is grabbing artwork from Amazon instead of using what's in the ID3 tag, or it's using cached artwork. A possible scenario: The track has no artwork, it's played, Synergy grabs the artwork for it and caches it somewhere, I add artwork to the track, I play it again, Synergy uses its cache instead of checking for updated artwork?
