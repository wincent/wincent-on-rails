Bug #195: iTunes sends no notification after finishing playlist

Kind bug
Product Synergy
When Created 2005-04-20T03:12:17Z, updated 2006-07-05T02:05:09Z
Status open
Reporter Steve Blunk
Tags no tags

Description

Since upgrading to 1.8.1 (I skipped 1.8), the center button in the menu bar display (I've tried the 3D and ultraPM2K5 sets) no longer switches from Pause to Play when a CD ends.

OS 10.3.8

Comments

  1. Greg Hurrell 2005-04-20T03:34:43Z

    Ah, interesting. I don't think this is specific to 1.8.1 (or 1.8). I suspect this happens in any recent version of Synergy when used with iTunes 4.7 or later, nor is it limited to when a CD finishes playing but rather will happen when any playlist comes to an end.

    This is a bug in iTunes. It should be sending a notification that its state has changed to "Stopped", but it doesn't. If you hover the mouse over the Synergy menu bar button you'll see from the tool tip that it thinks iTunes is still playing.

    I haven't got the retail version of Tiger, but I suspect that their iTunes widget will exhibit similar behaviour (unless they've implemented a workaround for it).

    So you'll see this behaviour when Apple fixes the bug, or when I come up with a workaround; not sure which will happen first.

    The iTunes widget Apple includes with Dashboard does indeed have the same problem.
