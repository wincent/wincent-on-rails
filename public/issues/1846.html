Bug #1846: Synergy shows incorrect album cover for different albums with same name and artist

Kind bug
Product Synergy
When 2011-07-28T01:13:08Z
Status open
Reporter anonymous
Tags no tags

Description

This is not exactly a bug, since it is caused by the way I name my classical music albums, and the fact that Synergy has presumably been developed with a different user in mind.

For example there are two albums named »Klaviersonaten (Badura-Skoda)« for which the artist is »Paul Badura-Skoda • pianoforte«. But the composers are different.

The first album of the two which plays creates a file: Artist-PaulBadura-Skoda•pianoforte,Album-Klaviersonaten(Badura-Skoda).jpg

And when a track from the other album plays, it shows the album cover for the first album. This not correct.

I understand that naming albums differently would solve the issue. However this is not practical in the iTunes interface.

Perhaps a solution would be for Synergy to add the composer to the cover image name if detected.

(This is unrelated to the problem in iTunes 10.4)
