Feature request #2208: Show issue change annotations using different styling

Kind feature request
Product wincent.com
When 2015-05-05T15:22:27Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

This was implemented the easy way, by synthesizing a comment. I don't really want to implement a new model or comment type for this, but I would like automated comments to be somehow styled differently. (More compact, more subtle?)

We're using wikitext right now, so that means we get:

Field changed:

  • From: foo
  • To: bar

I don't really want to go back and retroactively change these, but in an ideal world I'd like something more compact like:

Field: foo → bar

Throwing this up here as food for thought. I'll want to come up with something clean and minimal. Note that comments are taggable, so that might be the easiest way to mark these as special; one downside would be a heavier read load on the DB in order to render issue pages, and we'd wind up with a very important "synthetic" tag in the tag cloud (unless I added the option to mark tags as private, which may be the way of madness).
