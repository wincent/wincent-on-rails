Bug #1347: Symlink target links broken in git.wincent.com

Kind bug
Product wincent.com
When 2009-07-12T07:31:13Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

Just noticed this one. Try looking at a tree view which has symlinks in it, like this one, for example.

Note that you can click on the symlink names dot-vimrc etc, with no problem:

  • http://git.wincent.com/wincent.git?a=blob;f=dot-vimrc;h=1490f7ab0821b037ba088c07c62fb49964a83530;hb=21ad41f773c68061fb60f51f0aceb08b51f05654

But if you try to click on the target .vimrc itself:

  • http://git.wincent.com/wincent.git?a=object;f=.vimrc;hb=21ad41f773c68061fb60f51f0aceb08b51f05654

You end up getting redirected to a broken URL:

  • http://git.wincent.com/wincent.git/wincent.git?a=blob;f=.vimrc;h=af0bcf33b19955c6e75a95a19607312d5056a0dd;hb=21ad41f773c68061fb60f51f0aceb08b51f05654

Tried this on other Gitweb installs, eg this one and clicking on symlink targets seems to work fine (although that example is to a target inside a sub-directory).

I think I am going to try updating the version of Gitweb on the server and will see if that fixes it.
