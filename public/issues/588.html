Bug #588: synergy vs. last.fm - scrobbling conflicts

Kind bug
Product Synergy
When 2007-06-29T01:06:26Z
Status open
Reporter Jason Daniels
Tags no tags

Description

synergy scrobbling doesn't always recover if the last.fm scrobbling is opened and then closed again. I use synergy to scrobble most of the time, but sometimes bring up the last.fm app - to view an artist or my list of songs or to add the current song to my loved songs list. synergy seems to stop scrobbling when I do this, which makes sense, but it doesn't seem to start again if I quit last.fm.

Comments

  1. Greg Hurrell 2007-06-29T07:20:12Z

    Without better information, my guess is that on launching the other app, it initiates a new session and effectively renders the old one (the one started by Synergy) invalid.

    There might be a way for Synergy to detect this kind of temporary fault and correct it. The tricky part is distinguishing between temporary failures (which should be retried) and permanent ones (which should not be retried).

    Can you try turning on logging of Audioscrobbler events as described here:

    http://lists.wincent.com/pipermail/synergy-beta/2006-November/000003.html

    Then we can see what error messages are being returned by last.fm.

    So like I said, it may be possible to transparently work around this. In the absence of such a workaround the other possibility is to add something to the UI which allows the user to force a re-login (at the moment the only way to do this is clumsy; going into the prefs and making an arbitrary, termporary change to your login details to force Synergy to renegotiate).
