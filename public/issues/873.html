Feature request #873: Wiki tag search, tag#show, and (non-scoped) tag#search should sort by date

Kind feature request
Product wincent.com
When Created 2008-06-09T09:01:19Z, updated 2009-04-30T05:28:56Z
Status closed
Reporter anonymous
Tags no tags

Description

When a user searches for a tag on the wiki, the results are not sorted by any visible criteria, nor is there an option for user to sort.

Example: https://wincent.com/tags/red.hat?type=article

I suggest that results from a tag search be sorted by data, with newest to oldest. This presents users with the most up to date content at the top of the results list, which is probably the best default behaviour for the current implementation.
