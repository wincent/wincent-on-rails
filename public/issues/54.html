Bug #54: Synergy incorrectly prioritizes downloaded art over embedded art

Kind bug
Product Synergy
When Created 2004-12-05T10:33:26Z, updated 2009-11-29T12:31:42Z
Status open
Reporter Jeff Cannon
Tags no tags

Description

A feature that uses the artwork assigned by the iTunes user in the floater instead of the artwork found on the internet would be very handy. I take the time to go through and find album cover artwork for most of my music, and I find that a lot of the internet queries are wrong.

Being able to use the artwork that I have installed myself would be fabulous, even better would be a feature that uses user artwork as first priority, but if there is no artwork, then it searches the internet database.

Comments

  1. Greg Hurrell 2004-12-05T11:02:47Z

    Thanks for submitting the report. I know you phrased it as a request ("Show iTunes artwork instead of artwork found using the internet"), but I've reworded the summary because I actually qualify it as a bug (or at least a bad design decision).

    Synergy already does work very close to the way you request, so let me explain the details:

    1. Synergy first checks to see if there's any art in its cache and uses that if available. 2. If no art is available it asks iTunes. 3. If iTunes has no art it searches the Internet.

    It works this way because Synergy introduced support for album art before Apple released iTunes 4, the first version to support cover art. In other words, Synergy only ever used to use steps "1" and "3", and when iTunes 4 came out I added step "2".

    It usually works quite well, but it's not ideal. For example, if Synergy downloads the wrong art and

    • then* you add the correct art to iTunes, Synergy will continue using the wrong art, because that is

    what's in its cache. Solution is to delete the wrong art (click "Show Album Covers Folder" from the Synergy Global Menu when the pertinent track is playing and a Finder window will open with the incorrect cover selected; it's then fairly easy to just trash the file); Synergy will then grab the right art from iTunes.

    Needless to say, in Synergy Advance I haven't repeated the mistake of that 1-2-3 sequence above. Synergy Advance uses a plugin architecture and the user can rearrange the order of preferred album cover sources by drag and drop. Obviously by default it's good to have embedded art near the top of the source list, and things like Amazon search as a last resort.

    The challenge for Synergy Classic is how to redesign that 1-2-3 sequence without inducing too much of a performance hit (querying iTunes for art is pretty costly). It's also hard because I didn't design it with enough foresight. If you look at the version history you'll see how the program has grown in ways I never anticipated and didn't really prepare for (this is the whole reason I started writing a brand new app like Synergy Advance from scratch).
