Bug #173: 1.8 ignores cover art downloaded by previous versions if composer tag is set

Kind bug
Product Synergy
When Created 2005-04-13T23:31:13Z, updated 2005-04-18T16:17:17Z
Status closed
Reporter hiroshi eiyama
Tags no tags

Description

Hello. I'm using everyday this great software. I like it very much!

From version 1.8, Synergy has a nice feature that display composer's name in a floater. But that name is aiso included in album cover art file name (~/Application Support/Synergy/Album Cover), so now there is some album cover art file that is not displayed (and were nomally displayed at 1.7.1).

So I wish add a switch in preference for display "composer", or stop to include composer's name at album cover art file.

Currently I use previous Synergy 1.7.1. This is not a bug, but inconvinience for me.. Please fix it if you can.

Regards, eiyama

  1. Greg Hurrell 2005-04-14T00:45:50Z

    Thanks for submitting the report. This is definitely a bug but I hope it's not one that will cause too many problems. I've reworded the summary a little bit to better describe the issue.

    Version 1.8 won't "see" the art downloaded by previous versions because it now considers the composer (if set) to form part of the identifier of the track, therefore the identifiers have changed between versions.

    If you just wait a while it should re-download the art and use the new, updated name.

    The other alternative is to drag your art into iTunes and then Synergy will just use that.

    If other problems come up (such as picking the wrong art when it's re-downloaded) then I'll investigate ways to go back to the old behaviour. I'll keep this bug report open so people can report their experiences.

    Marking as ASSIGNED.

  4. Richard Brown 2005-04-17T06:27:30Z

    I think the track identifier and the floater contents should be independent of each other. Users may want to show the composer in the floater but this should not effect the track identifier and, consequently, the album art.

    The change to the track identifier potentially requires re-downloading all album artwork as well as probably multiplying the amount of album art cached on the hard disk (the same way that compilation albums have multiple copies of the same album art).

    I believe the best solution, if possible, would be to change the track identifier back to the pre-version 1.8 method but still display the composer information in the floater for those that want it.
