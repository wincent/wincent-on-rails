Feature request #307: REQUEST: Control another user's iTunes

Kind feature request
Product Synergy
When Created 2005-10-02T14:32:56Z, updated 2006-07-05T02:06:35Z
Status open
Reporter John Burgess
Tags no tags

Description

I'm probably crazy, but I have two accounts on my Mac - one for personal, one for business. I have my iTunes library stored under my personal account, but during the day, would like to play musci from my business account. How can I control one user's iTunes from another user?

This may not be something you want to "fix", but if you can tell me how to make it work, I'll be very happy.

Comments

  1. Greg Hurrell 2005-10-03T06:53:24Z

    Thanks for the request. There's no easy way of doing this, but it should be possible to implement. The basic problem is that the UNIX underpinnings of Mac OS X make it impossible for one user to control the processes of another user. Only the "superuser" (root) can do this.

    It's not appropriate for Synergy itself to run as root, but it should be possible to write a background daemon process that runs as root and starts at boot time. Synergy would issue instructions to the daemon and the daemon would then perform the necessary operations to effect changes in the processes of the other user.

    This is probably going to be fairly complicated, so it's unlikely to get implemented in the near future. Other, more oft-requested stuff will probably make it in first. But at least the request is in the database now so it won't be forgotten.
