Bug #390: Discrepancy between Growl and Floater text for Internet Radio

Kind bug
Product Synergy
When Created 2006-04-01T04:10:26Z, updated 2006-07-05T02:07:36Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

"Anyway, I'm using Synergy with Growl, and I notice that for some reason on the internet radio stations, it's not passing the correct artist and song name to the Growl notification system.

Using Synergy's built-in floater, it shows artist and song information fine, however using Growl in conjunction with Synergy will only show the correct radio station name, and not the artist and song info.

Is this a bug? Or am I doing something wrong? Thanks!"

My reply:

Thanks for the email. I wasn't aware of this issue. I doubt that you are doing anything wrong. I think the discrepancy may come from the way in which Synergy gets information from iTunes. I am not looking at the code right now, but from memory, the information in the Growl notifications is only that info which iTunes "pushes" out to observers using notifications. The information in the Floater, on other hand, is not only that "pushed" by iTunes but also "pulled" from it when Synergy polls it or explicitly asks for an update via Apple Events. That would explain the discrepancy.

In any case I will try to reproduce the problem here and see if I can get rid of the discrepancy. I think the reason that no one has noticed or reported this until now is that people generally don't use the Floater AND Growl; they normally use one OR the other and so don't notice the discrepancy.
