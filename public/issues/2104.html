Description

Hi Wincent, I love Command-T and use Vundle to install it and keep it updated. I am in a university network with proxy, that proxy doesn't support git protocol, and https://wincent.com/repos/command-t uses git protocol for the two submodules (i.e. "vendor/vimball" and "vendor/vimscriptuploader"). Is that possible for you to change the URL of submodules from git protocol to https protocol? That will make it much easier for people from a network with proxy. Thanks very much! --Frank

    Unfortunately there are people who are in the opposite situation (they have a Git build that uses the built-in protocol, and doesn't have HTTPS support built in).

    Those submodules aren't actually needed for the plug-in to run; they are just development-time dependencies. Is it possible for you to configure to Vundle to not do a recursive checkout? You don't actually need those submodules unless you want to run some of the Rake tasks that are used to do things like cut releases.
