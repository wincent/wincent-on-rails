Feature request #2211: Add support for directory-specific wildignore config

Kind feature request
Product Command-T
When Created 2015-06-10T17:35:49Z, updated 2015-07-02T19:28:47Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

I thought I had an issue already open for this somewhere, but anyway...

Not sure how general this should be, but the motivating use case here is to have a per-directory (or at least a per-project) setting for 'wildignore' (really, g:CommandTWildIgnore). This could be as general and flexible as checking for a settings file per-directory (or in the project root), or it could be more specific, and related only to 'wildignore' settings.

Motivating use case:

  • In the Corpus project, I generally want to ignore the "dist" directory
