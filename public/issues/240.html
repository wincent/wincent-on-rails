Feature request #240: REQUEST: Menu items for half-star ratings

Kind feature request
Product Synergy Advance
When Created 2005-05-17T01:09:12Z, updated 2007-05-17T01:50:51Z
Status closed
Reporter David Ameeti
Tags no tags

Description

I would like to be able to increase/decrease the rating for a song, but you only have options for setting explicit (full-star) ratings -- which means I can't set 3.5 stars, only 3 or 4. Even if the menu item didn't have a Cmd-key equivalent, that would be okay but that would be the easiest way to bump a rating up/down by half a star. I strongly prefer to not have to set a global keystroke that might interfere with other programs, just to bump a rating up/down by one. Another option instead: two menu items, one to add 1/2 star, one to decrease the same.

  1. Greg Hurrell 2005-05-17T01:24:34Z

    Thanks for submitting the request. Marking as ASSIGNED.

    I think a better idea would be to show half-star ratings only when holding down the option key. You'll notice that some of the menu items change depending on whether the option key is pressed and the same could be done for the items in the rating submenu. That way the feature you're asking for could be added without cluttering the menu item. What do you think of this idea?

    If I added items like you suggest it wouldn't be very convenient to increase a track's rating by 3 stars, would it? You'd have to open the menu, select increase rating, release the mouse button, open the menu, select, release, open, select, release...

    And to comment about not wanting to set global hot keys... There are a couple of things you can do to minimise interfering with other programs:

    1. In the Advanced preferences pane, select "Disable Synergy Advance hot keys when iTunes is not running".

    2. In one of the upcoming public preview releases I am going to add special exclusion/inclusion lists so that you can tell Synergy Advance to ignore hot keys when particular apps are frontmost, or conversely to only use hot keys when particular apps are frontmost. See feature request #11 for more details.
